Basic Usage

assume we have a collection called racers

Model - DAO class

assume we wanna create a DAO class implementing CRUD operations to Racers collection in mongodb, and store it in models/racer.rb

Completing ActionModel

  1. Add the ActionModel::Model mixin to the Racer class, make the class look like an ActionModel instance used by the Rails scaffold.
  1. Add instance methods to the Racer class.
  1. Using the Rails scaffold to generate controller

Notice:

The only concern here is the index controller get a bunch of racer hashes instead of instances of Racer. To fix this, we can add a helper method in helpers/racers_helper.rb.

Pagination

use will_paginate gem

  1. Add paginate class method to Model
  • accept a hash as input parameters
  • extract the :page property from that hash, convert to an integer, and default to the value of 1 if not set.
  • extract the :per_page property from that hash, convert to an integer, and default to the value of 30 if not set
  • find all racers sorted by number assending.
  • limit the results to page and limit values.
  • convert each document hash to an instance of a Racer class
  • Return a WillPaginate::Collection with the page, limit, and total values filled in – as well as the page worth of data.
  1. Change “racers#index” controller
  1. Add paginate to View app/views/racers/index.html.erb.
